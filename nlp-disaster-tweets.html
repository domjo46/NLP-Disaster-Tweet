Problem Statement¶

With growing technology use and the availability of social media, such as twitter, people are able to connect and communicate quicker than ever before.

The goal of this notebook is to classify whether a tweet (texts) is related to disaster event or not using deep learning methods.

Data Description¶

We are given about 10,000 tweets (sentences made from people) that were hand classified with a label of 1 for a real disaster and 0 otherwise. The data is split between training and test with about 7,613 samples for the training set and 3,263 samples for the testing set.

Below is a description of the features given

Columns Description
id (int) Unique ID for every tweet
keyword (text) Main word from the text, but this field may also be missing
location (text) Location the tweet was sent. May also be blank
text (text) The tweet itself
target {0,1} Boolean on whether the tweet is related to a disaster or not

Data Provider: Figure Eight Inc.

Original Data Source: Data For Everyone

Preprocessing¶

In [7]:
def clean_text(text): 
    
    # Lower case 
    clean = text.lower()
    
    # Remove punctuations
    clean = re.sub('['+string.punctuation+']','',clean)
    
    # Remove numbers as they are generic
    clean = re.sub('[\d+]','',clean)
    
    # Remove trailing white spaces
    clean = clean.strip()
    
    # Remove any double white spaces
    clean = re.sub('  ','',clean)
    
    # Remove these special characters from earlier
    clean = re.sub(r'[\x89\n\x9d\\]','',clean)
    
    return clean 

def preprocess(data, max_length=None, tokenizer=None): 
    data['clean_text'] = data['text'].apply(lambda x: clean_text(x))
    
    if not tokenizer: 
        tokenizer = Tokenizer() 
        tokenizer.fit_on_texts(data['clean_text'])
    
    sequences = tokenizer.texts_to_sequences(data['clean_text'])
    
    sequences = pad_sequences(sequences,padding='post',maxlen=max_length)
    
    return sequences, tokenizer.word_index, tokenizer

Word Embedding¶

In order to feed our text into our model, we have to transform our text into matrix form. We will be using word embeddings to transform our text data. The specific word embedding method we will be using here is GloVe embedding. GloVe embedding is efficient as it is pre-trained and is also good at retrieving the semantic relationship between words. On the down side though and particularly in this case, contextual meaning of words may be more important, but we will use this as our base and evaluate accordingly.

def get_glove_embeds(filename):
    embeddings = {}
    with open(filename, 'r', encoding='utf-8') as f:
        for line in f:
            values = line.split()
            word = values[0]
            vector = np.asarray(values[1:], dtype='float32')
            embeddings[word] = vector
    return embeddings

    axs.legend()

In order for us to combat the overfitting we attempt multiple methods but highlight the key differences between our initial and our final model:

  • Used an L1 regularizer, we also tested an L2 regularizer but found that the L1 regularizer performed better. This likely performed better because of the nature of our data. Specific words likely had more meaning and with L1 regularizer, it inherently feature selects or selects the more important words related to the sentiment of the tweet.

Results and Analysis¶

Our initial model proved to have very poor performance initially. We were able to see this through the diagnostic plots of loss vs epoch and accuracy vs epoch. In the loss vs epoch plot, we noticed as we increased the epoch, the training loss was decrease while the validation loss was increasing which was a clear sign of overfitting.

In order for us to combat the overfitting we attempted adjusting the hyperparameter. Difference between our final model and the our initial model highlights are below:

  • Decreased the learning rate to 1e-4 from 1e-3. Although the training data did not indicate the learning rate was too high, we adjusted the learning rate around to see which method performed best.
  • A dropout layer was added with 20% dropout. This helps by forcing the model to rely less on any specific set of parameters.
  • Used an L1 regularizer, we also tested an L2 regularizer but found that the L1 regularizer performed better. This likely performed better because of the nature of our data. Specific words likely had more meaning and with L1 regularizer, it inherently feature selects or selects the more important words related to the sentiment of the tweet.
  • Increased the number of epoch to ensure convergence of the loss

On the final model though, we still see evidence of overfitting as accuracy on the validation set tapers off after the fifth epoch with no further improvement.

Our final model was able to produce a prediction score of about 0.74 on the test set.

Conclusion¶

We were able to drastically improve an initial RNN model with GloVe embeddings on tweet data by tuning its hyperparameter. Our final model had a prediction score of 0.74.

Going forward to improve this model, we should look at further hyperparameter tuning of the model as there is still evidence of overfitting. Adjusting the model for complexity would likely only lead to more overfitting unless we add new layers such as drop out to help with this. Alternatively to get further improvement in this dataset, other models could be explored that are better fit to understand context such as BERT model.
